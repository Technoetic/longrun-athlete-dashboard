# Step 23 - Awwwards 디자인 패턴 분석

## Step-Back

실행 전에 먼저 답하라:
- 이 조사의 핵심 목적이 무엇인가? (한 문장)
- 이 조사 결과가 이후 어느 Step에 영향을 주는가?
- 조사에서 반드시 확인해야 할 핵심 항목 3가지는?

## 실행 내용

Step 22에서 수집된 스크린샷과 텍스트를 분석하여 디자인 패턴을 추출한다. **이 분석 결과가 이후 설계·구현의 유일한 디자인 기준이 된다.**

**필요한 파일:**

- `step_archive/screenshots/research/awwwards-*.png` (수집된 스크린샷)
- `step_archive/awwwards-*.txt` (수집된 텍스트)
- step016_조사결과_chunk*.md (프로젝트 특성 참조)

## 분석 방법

1. 수집된 스크린샷을 **모두** Read하여 디자인 패턴을 추출한다
2. 레이아웃/색상/간격/타이포그래피/인터랙션을 분석한다
3. **조사 축을 동적으로 생성**하여 분석한다

**조사 축 생성 규칙:**

- step016_조사결과_chunk*.md에서 프로젝트의 기능, UI 구성 요소, 대상 사용자를 분석하여 UX/UI·레이아웃에 영향을 주는 요소를 식별한다
- 식별된 요소마다 하나의 조사 축을 생성한다
- 사전에 고정된 축 목록을 사용하지 않는다

**사례 수 규칙:**

- 각 조사 축에서 관련 사례를 **최대한** 수집·분석한다
- **관련성 낮은 사례를 억지로 채우지 않는다** — 양보다 질
- 관련 사례가 1개뿐이면 1개만 기록하고, 그 이유를 명시한다

**각 조사 축에서 수행할 내용:**

- 수집된 Awwwards 사이트 중 해당 축과 관련된 참고 사례를 분석한다
- 각 사례의 레이아웃 패턴, UX/UI 패턴, 인터랙션, 시각적 구성을 기록한다
- 실현 가능한 대안을 최소 2개 이상 도출하고 장단점을 비교한다
- 특정 대안을 선택하지 않는다 — 비교 자료만 제공하고 결정은 기획 단계에서 한다

**조사 원칙:**

- 특정 기술/패턴/구조를 사전에 결정하거나 권장하지 않는다
- 조사결과에 명시되지 않은 요소를 임의로 추가하지 않는다
- 장점과 단점을 균형 있게 기술한다

## AI Slop 방지 체크리스트 (필수)

조사 결과에 다음을 **반드시 포함**한다.

1. **금지 패턴 식별** — 수집된 Awwwards 사례 중 다음 AI Slop 징후가 있는 사례는 "반면교사"로 별도 표시:
   - Inter/Roboto/Arial 산세리프 맹목 사용
   - 보라 계열 그라데이션 배경
   - 무조건 중앙정렬 카드
   - 과도한 border-radius
   - 획일적 단색 배경
2. **11가지 미학 축 매핑** — 수집 사례를 Brutalism / Glassmorphism / Minimalism(Swiss) / Dark OLED Luxury / Neumorphism / Cyberpunk 중 해당 축으로 분류. 각 축마다 물리적 CSS 제약(border 두께, shadow blur, grid 형태 등)을 구체 수치로 기록.
3. **토큰 추출** — 사례에서 관찰된 간격·타이포·색상 값을 4pt/8pt 그리드와 60-30-10 규칙에 맞게 정규화하여 기록.
4. **조립 가능성 평가** — 각 대안이 기존 디자인 시스템·Shadcn/ui 같은 컴포넌트 라이브러리로 조립 가능한지 여부.

## 결과 저장

**조사결과는 청크 단위로 저장한다:**

```text
step023_조사결과_chunk1.md (500줄 이하)
step023_조사결과_chunk2.md (500줄 이하)
step023_조사결과_chunk3.md (500줄 이하)
...
```text

**작성 규칙**:

- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다 (동시 실행 최대 10개).

서브에이전트는 sonnet을 사용한다.


## CoVe (Chain-of-Verification)

- [ ] 조사 결과가 청크 파일에 빠짐없이 저장되었는가?
- [ ] 이후 Step에서 참조할 수 있는 형식으로 정리되었는가?

미완료 항목이 있으면 해당 항목만 재조사한다.
