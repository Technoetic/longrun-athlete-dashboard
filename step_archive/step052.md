# Step 52 - 테스트 검증: semgrep 보안 결함 교차 분석

## 실행 내용

`semgrep --config p/security-audit src/` 실행하여 보안 결함이 테스트 실패의 원인인지 교차 분석한다. 보안 경고가 있는 코드 경로에서 테스트가 실패하면 보안 결함 우선 수정한다.

**분석 결과는 청크 단위로 저장한다:**

```text
step052_보안교차분석_chunk1.md (500줄 이하)
step052_보안교차분석_chunk2.md (500줄 이하)
...
```text

**작성 규칙**:
- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
