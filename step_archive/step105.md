# Step 105 - 부하 테스트: k6/Artillery 성능 측정

## Step-Back

실행 전에 먼저 답하라:
- 이 테스트의 핵심 목적은? (한 문장)
- 테스트 실패 시 어느 Step으로 돌아가야 하는가?
- 반드시 확인해야 할 엣지 케이스 2가지는?

프로젝트 특성과 트래픽 패턴을 분석하여 부하 테스트 시나리오와 성능 목표를 동적으로 결정한다.

k6 또는 Artillery를 사용하여 부하 테스트 및 성능 측정을 수행한다.
"웹 앱"이나 "API 서버"가 아니면 프로젝트 유형에 적합한 부하 테스트 도구와 방법을 선택한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다 (동시 실행 최대 10개).

**부하 테스트 단계에서 절대로 plan mode를 사용하지 않는다.**

**검증:**

- `.claude/hooks/load-test-validator.ps1`에서 자동 검증

**검증 실패 시:**

- 성능 병목 지점 분석
- 코드 또는 설정 최적화
- 검증 통과할 때까지 반복
