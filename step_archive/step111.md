# Step 111 - 수정 설계

## Step-Back

설계 전에 먼저 답하라:
- 이 설계가 해결해야 할 핵심 문제는? (한 문장)
- 조사/기획 결과에서 반드시 반영해야 할 제약 조건은?

Step 110 완료 후 plan mode에 진입하여 설계를 수행한다.

**필요한 파일:**

- step110_수정기획_chunk*.md (수정 기획 결과)

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용해 (동시 실행 최대 10개) 설계를 수행한다.

**설계 결과는 청크 단위로 저장한다:**

```text
step111_수정설계_chunk1.md (500줄 이하)
step111_수정설계_chunk2.md (500줄 이하)
step111_수정설계_chunk3.md (500줄 이하)
...
```text

**작성 규칙**:
- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.

## CoVe (Chain-of-Verification)

설계 완료 후:
- [ ] 조사/기획의 모든 요구사항이 설계에 반영되었는가?
- [ ] 구현 가능한 설계인가?
- [ ] 이전 실패 패턴을 반복하지 않는가?

## Self-Calibration

- 이 설계를 그대로 구현해도 되는가? (Y/N)
- N이면 해당 부분을 재설계한다.
