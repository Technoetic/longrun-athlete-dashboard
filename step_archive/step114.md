# Step 114 - 수정 회귀 테스트

## Step-Back

실행 전에 먼저 답하라:
- 이 테스트의 핵심 목적은? (한 문장)
- 테스트 실패 시 어느 Step으로 돌아가야 하는가?
- 반드시 확인해야 할 엣지 케이스 2가지는?

수정 구현 후 기존 기능이 정상 동작하는지 회귀 테스트를 수행한다.

## 테스트 범위

1. **기존 E2E 테스트 전체 재실행** — 수정으로 인해 기존 기능이 깨지지 않았는지 확인
2. **수정 관련 테스트** — 수정된 부분에 대한 새로운 테스트 케이스 추가 및 실행
3. **madge 순환 의존성 검증** — `npx madge --circular src/` 실행하여 수정 후 순환 의존성이 없는지 확인

## 검증 절차

1. `npx playwright test` 실행
2. 실패한 테스트가 있으면 원인 분석 후 수정
3. 모든 테스트 통과할 때까지 반복

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다 (동시 실행 최대 10개).

**이 단계에서 절대로 plan mode를 사용하지 않는다.**

서브에이전트는 항상 haiku를 사용한다.

## 수정 루프 가드레일 참조

step_archive/fix_guard.json을 확인하여:
- current_loop 값을 확인한다
- max_loops(3) 초과 시 사용자 개입 요청
- 동일 테스트 3회 연속 실패 시 스킵 처리
- 회귀 실패 시 step112(구현)만 재실행, step109로 돌아가지 않음

## Self-Calibration

테스트 완료 후:
- 모든 테스트가 통과했는가? (Y/N)
- Step-Back에서 정의한 엣지 케이스가 모두 커버되었는가? (Y/N)
- N이면 재실행한다.
