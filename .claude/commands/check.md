지금까지 대화에서 다룬 내용을 100% 이해했는지 점검한다.

**종료 조건: 보완·수정 작업이 단 한 건도 없이 100점이 나온 회차에서만 종료한다.**

---

## 핵심 원칙: 점검 에이전트 ≠ 판정 에이전트

같은 에이전트가 점검하고 스스로 통과 판정하는 것을 **금지**한다. 반드시 독립된 두 서브에이전트를 순차 스폰한다.

---

## 매 회차 실행 절차

현재 몇 번째 점검인지 표시하라.
예: `[점검 1회차]`, `[점검 2회차]`, ...

### Phase 1: 에이전트 A 스폰 (점검 전문)

Agent 도구로 서브에이전트를 스폰한다. 프롬프트:

```
너는 점검 전문 에이전트다. 아래 대화 요약을 바탕으로 0~3단계를 수행하고, 결과를 step_archive/이해_점검_결과.md에 저장하라. 최종 점수 판정은 하지 않는다.

[대화 내용 요약을 여기에 삽입]

## 0단계: 대화 요약
지금까지 대화에서 다룬 내용을 3~5줄로 요약하라.
요약에서 빠진 내용이 있다면 즉시 보완하고 **보완 발생: YES**로 기록한다.

## 1단계: 대화 주제 분류
- 개념형 / 코드형 / 절차형 / 복합형 중 해당하는 것을 선택하라.

## 2단계: 오해 감지 (선행 점검)
- 사실과 다르게 기억하고 있는 개념이 있는가?
- 혼동하기 쉬운 개념 쌍을 같은 것으로 착각하고 있지 않은가?
- 사용자가 정정하거나 강조한 부분을 반대로 기억하고 있지 않은가?
오해가 발견되면 즉시 정정하고 **보완 발생: YES**로 기록한다.
오해 1건당 -10점 패널티를 기록한다.

## 3단계: 항목별 점검
각 항목을 ✅ / ❌ / ⚠️(부분적)로 표시하라.

| 항목 | 기준 | 배점 |
|------|------|------|
| 핵심 개념 | 주요 개념과 용어를 정확히 설명할 수 있는가? | 20점 |
| 구조적 이해 | 전체 구조와 각 부분의 관계를 파악했는가? | 20점 |
| 맥락 파악 | 왜 이것이 필요한지, 어떤 문제를 해결하는지 아는가? | 20점 |
| 표면적 이해 | 내용을 그대로 재현할 수 있는가? | 20점 |
| 응용적 이해 | 새로운 상황에 변형·적용할 수 있는가? | 20점 |

❌ 또는 ⚠️ 항목이 있으면 즉시 보완하고 **보완 발생: YES**로 기록한다.

## 저장 형식
step_archive/이해_점검_결과.md에 다음을 저장하라:
- 0~3단계 전체 결과
- 보완 발생 여부: YES / NO
- 오해 패널티 합계: -X점
- .claude/에 파일 생성 금지
```

에이전트 A 완료를 기다린다.

### Phase 2: 에이전트 B 스폰 (판정 전문)

에이전트 A가 완료된 후, Agent 도구로 별도 서브에이전트를 스폰한다. 프롬프트:

```
너는 판정 전문 에이전트다. step_archive/이해_점검_결과.md를 읽고 4단계만 수행하라. 점검 내용을 수정하지 않는다.

## 4단계: 종합 결과
step_archive/이해_점검_결과.md의 내용을 독립적으로 검토하여 판정한다.

**종합 점수:** (기본 100점 - 오해 패널티 - 항목별 감점) / 100점

### 이 회차에 보완이 한 건이라도 발생했는가?

**YES → 100점이어도 종료하지 않는다. 즉시 다음 회차 시작을 메인 에이전트에 지시하라.**

**NO + 100점 → 진짜 종료.**
- 핵심을 한 문장으로 요약하라.
- 이 내용을 가르친다면 어떤 순서로 설명할지 제시하라.
- 대화 주제 유형에 맞는 다음 단계를 1가지 제안하라.
  - 개념형 → 실제 적용 사례
  - 코드형 → 테스트 또는 리팩터링 방향
  - 절차형 → 다음 단계 실행 항목
  - 복합형 → 가장 취약한 영역 보완 방향

**NO + 80~99점 → 다음 회차 시작을 메인 에이전트에 지시하라.**
**NO + 79점 이하 → 다음 회차 시작을 메인 에이전트에 지시하라.**

판정 결과를 출력하라.
```

### Phase 3: 반복 여부 결정

에이전트 B의 판정 결과를 확인한다.
- **보완 발생 YES 또는 100점 미만** → 즉시 다음 회차를 시작한다.
- **보완 발생 NO + 100점** → 종료한다.
