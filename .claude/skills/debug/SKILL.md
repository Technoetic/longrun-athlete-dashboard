# Debug Step 스킬

## 트리거 조건
- Step 제목에 "디버깅"이 포함될 때
- 테스트 실패 로그가 감지될 때
- 오류 메시지가 반복될 때

## 실행 절차

### 1단계: 오류 분류
1. 실패한 테스트/오류 메시지를 수집한다
2. 오류 유형을 분류한다:
   - **구문 오류** (SyntaxError, ReferenceError)
   - **런타임 오류** (TypeError, null reference)
   - **테스트 실패** (assertion failure, timeout)
   - **빌드 오류** (번들링, 의존성)

### 2단계: c8 커버리지 분석

- 미실행 라인/브랜치를 시각적으로 확인
- 테스트가 도달하지 못한 경로가 실패 원인과 관련될 수 있음

### 3단계: 서브에이전트 병렬 디버깅
- 오류 유형별로 독립 서브에이전트를 배정한다
- 각 서브에이전트는 haiku 모델을 사용한다
- 서브에이전트는 담당 오류만 수정하고 다른 코드를 변경하지 않는다

### 4단계: 검증
- 수정 후 전체 테스트를 재실행한다
- 실패 시 2단계로 돌아간다 (최대 3회)

## 금지 사항
- plan mode 사용 금지
- 오류와 무관한 코드 리팩토링 금지
- 테스트 케이스 삭제/비활성화 금지
